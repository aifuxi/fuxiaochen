# 前台站点 UI 重构设计说明

- 日期：2026-04-15
- 范围：前台站点（不含 CMS 后台）
- 依据：[DESIGN_SPEC.md](/Users/fuxiaochen/workspace/fuxiaochen/docs/DESIGN_SPEC.md)
- 重构目标：以视觉重构为主，尽量保留现有信息架构和功能行为

## 1. 背景与目标

当前前台站点已经具备深色、高对比、衬线标题与技术感标签等初步基底，但仍混用了较多与 `Chen Serif` 规范冲突的视觉表达，例如：

- 玻璃质感卡片
- shimmer / glow / spotlight 等装饰性特效
- 依赖 blur 建立层级
- 偏 showcase 风格的大图卡片与浮起交互

这些元素会削弱规范中强调的 editorial 气质、结构秩序和内容优先原则。

本次重构的目标是：

- 让前台首页、列表页、详情页、介绍页在视觉语法上统一
- 把“高级感”的来源从特效转移到排版、比例、边界与留白
- 保留现有内容模块与功能路径，避免把本次任务扩大成信息架构重做
- 为后续 CMS 与前台统一设计系统打下可复用基础

## 2. 已确认的设计结论

### 2.1 重构范围

本次只改造前台站点，覆盖以下页面和共享区块：

- `/`
- `/articles`
- `/article/[slug]`
- `/projects`
- `/about`
- `/friends`
- `/changelog`
- `SiteHeader`
- `SiteFooter`
- 文章卡片、文章行列表、项目卡片、订阅区、正文排版相关样式

CMS 后台不在本轮范围内。

### 2.2 改造深度

本次采用“视觉重构为主”的策略：

- 保留现有页面的大体模块顺序和核心信息结构
- 允许对页面内部的排版节奏、容器样式、间距和区块层次做重构
- 不改动前台功能逻辑、数据接口和主要交互路径

### 2.3 方向选择

前台重构采用 `A / 严格 Chen Serif` 作为主方向，并吸收少量现有站点的人物与作品展示感。

这意味着最终界面会：

- 更接近“高端数字出版物”
- 明显弱化 showcase 模板感
- 保留适度的人物/项目露出，避免过度冷感

## 3. 视觉语言

### 3.1 总体气质

前台应传达以下印象：

- 冷静
- 精确
- 克制
- 有编辑感
- 高级但不浮夸

整体风格不追求赛博、霓虹或科技炫技，而应像一套经过编辑和校对的数字刊物系统。

### 3.2 视觉主轴

界面的高级感主要由以下因素建立：

- `Newsreader` 标题比例
- `Inter` 正文可读性
- `Space Grotesk / Mono` 标签与系统信息
- 深色实体表面
- hairline border
- 清晰的分区结构
- 节奏稳定的留白

### 3.3 明确移除的表达

前台所有共享组件与页面样式中，应移除或停止依赖以下表达：

- `glass-card`
- `shimmer-border`
- `spotlight-card`
- `spotlight-border`
- 大面积 blur/glow 背景
- 鼠标聚光跟随
- 持续 pulse / shimmer / 浮动类装饰动画
- 以 hover 抬升和投影为主的“展示卡片”交互

这些效果即使局部保留，也不能继续承担主要层次构建职责。

### 3.4 替代策略

以上效果应统一替换为更克制的结构化表达：

- 用深色 solid surface 替代玻璃感背景
- 用细描边和分隔线替代发光边界
- 用标题、摘要、元信息和栅格比例建立层次
- 用留白、列宽和模块顺序建立页面节奏
- 用局部强调色和当前态标识替代大面积强调块

## 4. 设计 Token 与使用约束

### 4.1 颜色

前台以中性色为主体：

- 基础背景：`#050505`
- surface：`#0B0B0B` / `#111111` / `#161616`
- 主文本：`#EBEBEB`
- 次文本：`rgba(235,235,235,0.72)`
- 第三级文本：`rgba(235,235,235,0.48)`
- 强调色：`#10B981`

使用原则：

- 强调色仅用于当前导航、分类标签、关键数字、链接焦点和小范围重点信息
- 不使用大面积 emerald 铺底
- 页面层次优先通过灰阶与边界建立，而不是通过彩色背景建立

### 4.2 字体

- 大标题、栏目标题、文章正文标题：`Newsreader`
- 正文、摘要、说明性文本：`Inter`
- 标签、日期、统计、导航、小型 meta：`Space Grotesk` / mono token

应用原则：

- serif 负责品牌和 editorial 气质
- sans 负责阅读效率
- mono 负责系统感和结构信息
- 不让 mono 大面积占据正文主阅读区域

### 4.3 圆角、边界与阴影

- 卡片、容器采用中到大圆角，但需克制
- 边界使用细线，优先 `rgba(255,255,255,0.06~0.14)`
- 阴影仅作为弱辅助，不作为主要层次手段
- 不使用发光阴影和模糊外溢营造“高级感”

## 5. 页面骨架与共享模式

### 5.1 统一页面骨架

前台页面统一到一套出版物式页面骨架：

1. 顶部导航
2. 页面标题区 / Hero 区
3. 主内容区
4. 相关内容 / 次级区块
5. 页脚

不同页面的差异主要体现在内容密度和版式比例，不体现在视觉语法切换。

### 5.2 页面标题区模式

首页之外的前台页面，应尽量统一为同一套标题区表达：

- mono 栏目标签
- serif 大标题
- 一段简短导语
- 必要时附加统计信息或栏目说明

目标是让 `/articles`、`/projects`、`/about`、`/friends`、`/changelog` 都具备一致的栏目感与归档感。

### 5.3 分区方式

页面之间和模块之间的层次优先用以下方式建立：

- 纵向留白
- 容器宽度差异
- 细分隔线
- 标题层级
- 列表节奏

避免依赖“一个区块一个花样”的方式建立差异。

## 6. 关键页面设计方案

### 6.1 首页 `/`

首页继续保留现有模块：

- Hero
- 精选文章
- 最新文章
- 作者/简介区
- 精选项目
- Newsletter

但视觉逻辑重构如下：

- Hero 改为 editorial 首屏，人物图像退居辅助位置
- 首屏不再使用漂浮标签、玻璃角标和强装饰性按钮
- 精选文章与项目区块采用更统一的标题区和卡片边界语言
- 最新文章更像 archive/list，而不是 hover showcase 列表
- 作者区从“名言 + 发光卡片”改为更安静的编辑者介绍块
- Newsletter 区改成“订阅编辑部通讯”的出版物式模块，不做营销感大 CTA

首页的重点不再是“展示设计效果”，而是“建立站点内容气质”。

### 6.2 文章列表 `/articles`

文章列表页保持归档属性，增强以下特征：

- 更清晰的页面标题区
- 更稳定的列表节奏
- 分类、篇数、分页等 meta 信息使用 mono 统一表达
- 列表项弱化图片存在感，强化标题、摘要与发布时间

该页应更像“可浏览的文章归档页”，而不是卡片画廊。

### 6.3 文章详情 `/article/[slug]`

文章详情页会向文档/出版物体验靠拢：

- 顶部封面弱化为辅助内容，不承担全部视觉重量
- 标题、作者、分类、日期、阅读时长形成独立的文章头部信息区
- 正文容器宽度、字号、段落间距与 heading 节奏重新校准
- 目录侧栏改为更安静的文档目录盒，不使用玻璃和浮层感
- 相关文章区继续保留，但视觉语言与文章卡片保持一致

该页的设计核心是阅读体验，而不是首屏展示效果。

### 6.4 项目页 `/projects`

项目页保留作品展示能力，但降低作品集模板感：

- 页面标题区与文章页保持统一栏目语法
- 项目统计区改为更结构化的数字摘要块
- 项目卡片减少封面图压迫感，突出标题、说明、技术栈和成果信息
- 项目入口的行动文案保持克制，不做夸张 CTA

页面应表达“有判断力的工作记录”，而不是“强调视觉冲击的作品墙”。

### 6.5 关于页 `/about`

关于页保留人物介绍、技能、经历等信息，但整体转为出版物式个人介绍页：

- 首屏更像作者页，而不是带浮层装饰的人像展示页
- 技能、装备、经历这些内容改为统一的结构化列表/卡片语言
- 去掉装饰性图层、blur 和 reveal 噱头
- 保持人物温度，但避免页面每个区块视觉语言都不同

### 6.6 友链与更新日志

这两类页面同样要纳入统一骨架：

- 使用相同的标题区
- 列表与条目沿用统一边界和 meta 样式
- 强调信息排序、时间感和浏览效率

## 7. 共享组件改造原则

### 7.1 `SiteHeader`

导航重构目标：

- 更安静、更稳定
- 保留 fixed/sticky 能力
- 去掉 blur-heavy 和强“悬浮条”观感
- 使用边线、背景透明度和当前态文本颜色建立状态变化

交互上：

- 保留吸顶状态切换
- 去掉 logo 旋转、过强 hover underline 动效和 glow 按钮
- CTA 按钮改成更克制的实体或描边按钮

### 7.2 `SiteFooter`

页脚改为更简洁的出版物式尾部：

- 品牌标识简化
- 文本与版权信息排版更整洁
- 与正文形成明确收束，而非仅作为最底部一条信息栏

### 7.3 `ArticleCard`

文章卡片改造目标：

- 降低 showcase 感
- 增强内容卡属性
- 使用清晰边界、标题层级和 meta 信息表达阅读价值

具体上：

- 弱化 glass / shimmer / spotlight
- 减少 hover 浮起幅度
- 封面图只做辅助，不抢标题
- 类别、日期、阅读时长形成稳定信息区

### 7.4 `ArticleRow`

文章行列表应更像 archive item：

- 边界清晰
- hover 只做轻微背景或文字强调
- 箭头动效降噪
- 保证移动端下的纵向阅读效率

### 7.5 `ProjectCard`

项目卡片从“视觉海报”调整为“案例摘要卡”：

- 图片压暗和大面积覆盖层弱化
- 让标题、描述、技术栈和指标信息成为主体
- 用结构化内容替代强视觉覆盖

### 7.6 `NewsletterCard`

订阅区改为更符合编辑部语境的模块：

- 视觉上更像一段刊物订阅区，而非营销 landing CTA
- 去掉大 blur 光团和 glow button
- 强调表单可读性、信任感和低干扰

## 8. 动效与状态策略

### 8.1 保留的动效

允许保留的动效仅限于：

- 页面或区块进入时的轻微淡入
- 导航吸顶时的克制过渡
- 链接、按钮、卡片 hover 的轻量颜色/边界变化
- 极轻的位移或透明度反馈

### 8.2 禁止继续使用的动效

- shimmer
- spotlight 跟随
- 持续 pulse 装饰动画
- 大幅 hover 抬升
- 长时间吸引视线的循环装饰动效

### 8.3 状态表达

所有状态优先通过以下方式表达：

- 文本颜色变化
- 边界颜色变化
- 弱底色
- 局部点状标识
- 当前栏目 underline 或 marker

而非通过发光、模糊和剧烈动画表达。

## 9. 实现边界

本次实现不包含：

- CMS 后台样式统一
- 新增前台功能模块
- 文章、项目、关于页的信息架构大改
- 新的数据模型或接口调整

本次实现包含：

- 全局样式 token 与公共 utility 收敛
- 前台共享 layout / block / ui 样式重构
- 前台核心页面的视觉统一
- 正文排版和目录样式优化

## 10. 验收标准

重构完成后，应满足以下结果：

- 前台已基本移除与规范冲突的 glass / glow / shimmer / spotlight 语言
- 首页、列表、详情、项目、关于页具备一致的视觉语法
- 页面阅读体验优先于展示效果
- 强调色使用克制，不再大面积主导界面
- 导航、卡片、列表、目录、订阅区都能看出属于同一套系统
- 移动端与桌面端都保持良好可读性与版式稳定性

## 11. 实施建议

建议实现顺序：

1. 全局 token、utility 和前台共享 layout 收敛
2. Header / Footer / 公共卡片 / 列表项 / Newsletter 统一
3. 首页重构
4. 文章列表与文章详情页重构
5. 项目页、关于页、友链页、更新日志页补齐
6. 统一验收与样式清理

该顺序有利于先建立系统，再逐页替换，降低重复返工。
