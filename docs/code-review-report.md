### P1 - High (4 issues)

#### 1. `dangerouslySetInnerHTML` XSS 风险
**Location**: [app/(public)/article/[slug]/page.tsx:239](app/(public)/article/[slug]/page.tsx#L239)

文章内容直接使用 `dangerouslySetInnerHTML` 渲染 HTML。当前是 mock 数据，如果将来接入真实 CMS 数据，需要确保内容经过 sanitization（如 DOMPurify）。

**建议**:
```tsx
import DOMPurify from 'dompurify';

// 使用前 sanitization
const sanitizedContent = DOMPurify.sanitize(content);
<div dangerouslySetInnerHTML={{ __html: sanitizedContent }} />
```
