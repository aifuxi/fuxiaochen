# Changelog 页面重构设计

## 概述

重构前台 changelog 页面，采用简洁、清晰的卡片列表布局，符合 Apple Human Interface Guidelines。

## 设计目标

- 简单清晰的视觉呈现
- 符合 Apple Human Interface Guidelines 设计规范
- 更现代的无限滚动加载方式

## 布局结构

```
┌─────────────────────────────────────────────────┐
│  更新日志                                         │
│  记录产品的每一次迭代与改进                         │
├─────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────┐│
│  │ v1.2.0        ·        2024年2月20日         ││
│  │                                             ││
│  │ 更新内容（Markdown 渲染）                     ││
│  │ - 新功能 A                                   ││
│  │ - 修复 B                                     ││
│  └─────────────────────────────────────────────┘│
│                                                 │
│  ┌─────────────────────────────────────────────┐│
│  │ v1.1.0        ·        2024年2月15日         ││
│  │                                             ││
│  │ 更新内容...                                  ││
│  └─────────────────────────────────────────────┘│
│                                                 │
│  [加载更多...]                                   │
└─────────────────────────────────────────────────┘
```

## 设计细节

### 1. 页面头部

- 标题：更新日志
- 副标题：记录产品的每一次迭代与改进

### 2. 卡片设计

- 背景：`bg-surface`
- 边框：`border border-border`
- 圆角：`rounded-xl` (20px)
- 内边距：`p-6`
- 第一张卡片添加「最新」徽章（`bg-accent`）

### 3. 卡片头部

- 版本号：`font-semibold text-lg text-text`
- 分隔符：`·` 带淡色
- 日期：`text-sm text-text-secondary`

### 4. 无限滚动

- 首次加载 10 条
- 使用 Intersection Observer 检测滚动到底部
- 加载时显示 loader
- 加载完毕无更多数据时显示「已加载全部」

### 5. 空状态

- 居中显示「暂无更新日志」

## 技术实现

- 保留现有的 `getChangelogsAction` Server Action
- 使用 SWR 的 `useSWRInfinite` 实现无限滚动
- 使用 `IntersectionObserver` 触发加载更多
- 保留 `BlogContent` 组件渲染 Markdown 内容
