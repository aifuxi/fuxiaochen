# Chen Serif 完整设计规范

## 1. 文档定位

这是一份完整、独立、可跨项目复用的设计规范文档。

它的目标不是描述某一个页面，而是定义一套可稳定迁移到不同项目中的设计系统，包括：

- 品牌气质
- 视觉原则
- Design Tokens
- 组件规范
- 页面模式
- 交互状态
- 文案语气
- 复用策略
- 项目接入流程

阅读这份文档时，不需要依赖任何其他说明文件。它应当可以单独作为：

- 设计师的视觉基线
- 前端工程师的实现依据
- 产品经理的页面结构参考
- 新项目的设计系统接入手册

## 2. 系统概览

### 2.1 系统名称

`Chen Serif`

### 2.2 系统定位

`Chen Serif` 是一套偏冷静、克制、精确的数字产品设计系统。它不依赖炫技特效来制造“科技感”，而是通过排版、比例、留白、结构分层和低干扰动效，建立一种带有 editorial 气质的高端数字产品体验。

它同时服务两个目标：

1. 保留明显的品牌气质与审美判断。
2. 在产品页面、后台、内容页和流程页中保持稳定、清晰、可预测的使用体验。

它适合那些希望同时拥有：

- 品牌感
- 文档感
- 工具感
- 可复用性

的项目。

### 2.3 一句话定义

这是一套通过“排版 + 结构 + 边界 + 克制交互”建立高级感的设计系统，而不是一套依赖玻璃、流光和强视觉噱头的风格包。

## 3. 设计原则

### 3.1 以排版建立气质，而不是以特效建立气质

界面的高级感主要来自：

- 标题比例
- 正文节奏
- 模块间距
- 边线秩序
- 留白控制

而不是来自：

- 毛玻璃
- 流光边框
- 鼠标聚光
- 霓虹外发光
- 强装饰动画

### 3.2 品牌感优先体现在阅读体验中

用户首先感知到的应该是：

- 字体选择
- 内容语气
- 结构节奏
- 信息编排
- 页面秩序

而不是单一视觉效果。

### 3.3 产品界面必须可操作、可复用、可扩展

无论是：

- 官网
- 博客首页
- 文章详情页
- Dashboard
- 设置页
- 表单流程

都应共享同一套视觉语法和交互状态模型，避免品牌页和产品页看起来像两个系统。

### 3.4 用克制建立识别度

这套系统的识别度不来自“更多设计”，而来自“更少但更准确的设计”：

- 颜色数量少
- 强调色面积少
- 动效数量少
- 装饰元素少

但每一处强调都必须有清晰目的。

### 3.5 结构优先于装饰

卡片、分隔线、列表、导航关系、表单分组、目录、归档、状态区，这些结构性元素的优先级必须高于装饰型图形。

装饰只能辅助结构，不能替代结构。

## 4. 品牌气质

### 4.1 关键词

- 冷静
- 精确
- 克制
- 有判断力
- 编辑感
- 高级但不浮夸
- 现代但不炫技

### 4.2 应传达的印象

- 像一份被认真编辑过的数字出版物
- 像一套经过推敲的专业工具
- 像一个有明确审美判断的高端产品

### 4.3 不应传达的印象

- 过度赛博朋克
- 霓虹感过强
- 强营销口号化
- 普通 SaaS 模板感
- 依赖透明材质和光效制造“伪高级感”

## 5. 适用范围

### 5.1 强匹配项目

- 技术写作平台
- 个人博客
- 知识库
- 开发者工具
- 内容平台
- SaaS 控制台
- 数据后台
- 结构化创作工具

### 5.2 中匹配项目

- 品牌官网
- 企业官网
- AI 工具产品页
- 内容媒体站

### 5.3 弱匹配项目

- 娱乐型产品
- 社交型产品
- 儿童产品
- 依赖强情绪、大面积高饱和视觉和大量装饰动画的产品

### 5.4 页面类型覆盖

这套系统应覆盖：

- 官网 / 品牌页
- 博客首页
- 文章详情页
- 文档页
- 列表页
- 归档页
- Dashboard / 控制台
- 设置页
- 表单流程页
- 登录 / 注册 / 验证等入口页

## 6. 核心视觉方向

### 6.1 应保留的特征

- 黑底白字的高对比主视觉
- `Newsreader` 带来的 editorial 标题气质
- `Inter` 带来的正文可读性
- `Space Grotesk` 带来的技术标签与系统感
- 少量 `#10B981` 作为局部强调色
- 大圆角、细描边、清楚的层级分区
- 克制且有重量感的动效

### 6.2 必须移除的特征

- Glassmorphism / 毛玻璃
- Shimmer Border / 流光边框
- Spotlight Cursor Tracking / 鼠标聚光跟随
- 大面积 Glow / 发光外溢
- 依赖 Blur 建立主要层次
- 强视差
- 持续浮动
- 长时间吸引注意力的装饰动画

### 6.3 替代方向

- 用 solid surface 取代玻璃材质
- 用 hairline border 取代动态边框
- 用结构性背景平面取代交互光效
- 用列表、归档、目录和版式节奏建立文档感

## 7. Design Tokens

### 7.1 颜色 Color

#### 基础背景

- `bg/base`: `#050505`
- `bg/surface-1`: `#0B0B0B`
- `bg/surface-2`: `#111111`
- `bg/surface-3`: `#161616`

#### 边界与分隔

- `line/subtle`: `rgba(255,255,255,0.06)`
- `line/default`: `rgba(255,255,255,0.08)`
- `line/strong`: `rgba(255,255,255,0.14)`

#### 文本

- `text/primary`: `#EBEBEB`
- `text/secondary`: `rgba(235,235,235,0.72)`
- `text/tertiary`: `rgba(235,235,235,0.48)`
- `text/inverse`: `#050505`

#### 强调与状态

- `accent/emerald`: `#10B981`
- `accent/emerald-muted`: `rgba(16,185,129,0.16)`
- `state/error`: `#EF4444`
- `state/warning`: `#F59E0B`
- `state/success`: `#10B981`

#### 使用规则

- 大面积背景只使用黑、深灰、白这组中性色。
- `emerald` 只用于关键数字、当前状态、焦点态、选中态、小标签和局部强调。
- 不允许把 `emerald` 用作大面积铺底色。
- 后台列表、表格、设置页和表单中，强调优先通过文字、边线、点状标识和微弱底色表达，而不是整块高饱和背景。

### 7.2 字体 Typography

#### 字体家族

- `font/display`: `Newsreader`
- `font/body`: `Inter`
- `font/mono-label`: `Space Grotesk`

#### 字号层级

- `display/xl`: `72-96px`, `line-height: 0.92`
- `display/lg`: `56-72px`, `line-height: 0.95`
- `heading/1`: `40px`, `line-height: 1`
- `heading/2`: `32px`, `line-height: 1.05`
- `heading/3`: `24px`, `line-height: 1.1`
- `title/lg`: `20px`, `font-weight: 500`
- `title/md`: `18px`, `font-weight: 500`
- `body/lg`: `16px`, `line-height: 1.6`
- `body/md`: `15px`, `line-height: 1.55`
- `body/sm`: `14px`, `line-height: 1.5`
- `label`: `11px`, uppercase, `letter-spacing: 0.18em`
- `micro`: `10px`, uppercase, `letter-spacing: 0.16em`

#### 字体分工

- 主标题和关键区块标题使用 `Newsreader`
- 正文、说明、帮助文案使用 `Inter`
- 表头、标签、元信息、状态标签优先使用 `Space Grotesk`

#### 使用规则

- italic 只允许用于单词级或短语级强调
- 不建议整段使用 italic
- 后台页面可以保留少量 serif 标题，但字段标签、数据表头、说明信息必须回归 sans / mono

### 7.3 间距 Spacing

采用 4px 基线系统：

- `4`
- `8`
- `12`
- `16`
- `20`
- `24`
- `32`
- `40`
- `48`
- `64`
- `80`
- `96`

#### 使用规则

- 卡片默认内边距：`24px` 或 `32px`
- 大 section 间距：`64px - 96px`
- 表单字段垂直间距：`20px - 24px`
- 后台高密区域允许下降到 `16px`

### 7.4 圆角 Radius

- `10px`
- `16px`
- `24px`
- `32px`
- `999px`

#### 使用规则

- 输入框、下拉、按钮默认 `16px`
- 卡片默认 `24px`
- 关键展示区块可用 `32px`
- 不使用普通 SaaS 常见的 `6px / 8px` 小圆角

### 7.5 边框与分隔 Border & Divider

- 默认边框：`1px solid rgba(255,255,255,0.08)`
- 强调边框：`1px solid rgba(255,255,255,0.14)`
- 分隔线：`1px solid rgba(255,255,255,0.06)`

#### 使用规则

- 优先使用细边线建立层次
- 不依赖透明材质和重阴影做分层
- 卡片、表格、侧栏、弹层、表单分组应共享统一的 divider 逻辑

### 7.6 阴影 Shadow

- 常规卡片：`0 6px 20px rgba(0,0,0,0.22)`
- 浮层 / 菜单：`0 12px 30px rgba(0,0,0,0.32)`
- Hover / CTA：`0 8px 18px rgba(0,0,0,0.24)`

#### 使用规则

- 阴影只用于建立贴地感与轻微层次
- 不使用彩色阴影
- 不使用大面积发光阴影

### 7.7 动效 Motion

#### 基础参数

- 基础曲线：`cubic-bezier(0.16, 1, 0.3, 1)`
- `fast`: `140ms`
- `base`: `220ms`
- `slow`: `360ms`

#### 允许的动效

- hover 上浮 `2-4px`
- 边框颜色变化
- 文本或图标颜色变化
- section 渐入上移
- modal / drawer 淡入与轻微位移
- step flow 状态切换

#### 禁止的动效

- shimmer
- pulse glow
- cursor tracking
- 强视差
- 长时间循环且无功能价值的装饰动画

## 8. 组件规范

### 8.1 Button

#### 类型

- `Primary`
- `Secondary`
- `Ghost`
- `Destructive`

#### 规范

- 默认高度：`44px`
- 大按钮：`52px`
- 紧凑按钮：`36px`
- 默认圆角：`16px`
- 主 CTA 可使用 `pill`
- 水平内边距：`16px - 20px`
- 字体：`Inter`, `14-15px`, `font-weight: 500`

#### 状态规则

- hover：轻微上浮、背景对比提升或边框增强
- focus-visible：细 ring + 边框强化
- active：轻微下压或轻度缩放
- disabled：降低对比，不改变布局

#### 禁止事项

- 强 glow
- 夸张缩放
- 果冻感动画
- 过重阴影

### 8.2 Input / Textarea / Select

#### 基础规范

- 默认高度：`44px`
- 背景：`surface-1` 或 `surface-2`
- 边框：`line/default`
- placeholder：`text/tertiary`
- 输入文字：`text/primary`
- label 位于控件上方
- label 使用 `Space Grotesk`, `11px`, uppercase
- 帮助文案与错误文案使用 `Inter`, `13-14px`

#### Textarea

- 最小高度：`120px`
- 文本顶对齐
- 不使用重内阴影

#### Select / Combobox

- 右侧图标区域保持简洁
- 菜单弹层与输入框共享同一套 surface、border、radius 语言

#### 状态规则

- `default`：弱边框、低对比底色
- `hover`：边框略增强
- `focus`：边框增强 + 极淡 emerald focus ring
- `error`：边框与提示转为 error
- `disabled`：降低可用性对比但保持可读

### 8.3 Form Layout

#### 规则

- 默认使用单列表单
- 宽屏下允许双列，但只用于逻辑独立字段
- 字段组之间间距：`24px`
- 大 section 间距：`40px - 48px`
- 每个 section 应包含标题、可选说明、字段区
- section 标题可少量使用 `Newsreader`
- 字段级标题不使用 serif

#### 多步骤流程

- stepper 使用 `Space Grotesk`
- 当前步骤可使用细线或 emerald 点标识
- 不使用大色块进度条
- 主按钮优先级保持稳定

### 8.4 Card

#### 类型

- `Editorial Card`
- `Data Card`
- `Form Section Card`
- `Overlay Card`

#### 规范

- 背景：`surface-1`
- 边框：`1px line/default`
- 默认圆角：`24px`
- 内边距：`24px` 或 `32px`
- 标题与内容之间保持明确层级
- hover 只允许轻微上移和边框增强

#### 禁止事项

- 发光边框
- 动态描边
- 鼠标跟随高亮
- 玻璃糊底

### 8.5 Table / List

#### 规范

- 表头：`Space Grotesk`, `10-11px`, uppercase
- 行高：`48px - 56px`
- 单元格文本：`Inter 14-15px`
- 数字列右对齐
- 行之间使用细分隔线
- hover 行只做低对比底色提升
- selected row 用左侧细线、边框或弱底色表达

#### 规则

- 状态列使用 text + dot / chip
- 不使用整行染色表达状态
- 数据密集页可使用 sticky header 和弱 zebra 分层，但必须保持安静整洁
- 列表项优先使用标题 + 摘要 + metadata 的清晰结构

### 8.6 Tag / Badge / Status

#### 规范

- 高度：`20px - 24px`
- 字体：`Space Grotesk` 或 `Inter 12px`
- 默认形式：弱底 + 细边框
- success / warning / error 只做小面积状态区分

#### 规则

- 用于状态、分类、主题标记
- 保持轻量
- 不用于承载长句

### 8.7 Navigation

#### 官网导航

- 文本链接为主
- 当前项用细下划线或文字增强表示
- CTA 保持克制，不应喧宾夺主

#### 后台导航

- 可使用侧栏或顶部导航
- 当前项通过左侧细线、弱底色或文字增强体现
- 分组标题使用 `Space Grotesk`
- 图标细、简洁、几何化

### 8.8 Modal / Drawer / Popover

#### 规范

- 使用深色 solid surface
- 边框清晰
- 阴影克制
- radius：`24px`
- overlay 使用深色透明遮罩，不做 blur
- modal、drawer、popover 共享统一 framing 语言

### 8.9 Empty / Loading / Feedback

#### Empty State

- 结构：标题 + 说明 + 主行动 + 可选次行动
- 图形优先极简线框或结构示意
- 文案需说明为何为空、下一步做什么

#### Loading

- 按钮 loading：保留尺寸，用 spinner + 文案
- 区块 loading：使用 solid skeleton block
- 表格 loading：保留表头与结构，内容区 skeleton

#### Feedback

- success、warning、error 都应明确但克制
- 优先用图标、标题、短说明和边框区分，而不是大色块

## 10. 交互状态规范

### 10.1 全局状态模型

所有交互组件尽量遵循统一状态集：

- `rest`
- `hover`
- `focus-visible`
- `active`
- `selected`
- `disabled`
- `loading`
- `error`
- `success`

### 10.2 统一规则

- 状态变化优先通过边框、底色、文字对比、轻微位移表达
- 不使用 glow、闪烁、强缩放、复杂动画
- 同类组件的状态语言必须一致

### 10.3 Hover

- 按钮：轻微上浮 `2px`
- 卡片：边框略增强 + 上移 `2-4px`
- 列表行：底色略提升
- 链接：underline 展开或文字颜色变化
- hover 只作为桌面端增强，不承载关键语义

### 10.4 Focus Visible

- 键盘焦点必须清晰可见
- 默认用边框增强 + 极淡 emerald ring
- 使用 `focus-visible`
- 所有可交互元素都需支持统一焦点样式

### 10.5 Active

- 按下态只确认“操作已触发”
- 允许轻微下压 `1px` 或缩放到 `0.99`
- 不使用剧烈视觉反馈

### 10.6 Selected

- 优先使用细边线、左侧竖线、弱底色、文字增强表达
- `emerald` 只做局部标识
- 不使用高饱和整块 selected 背景

### 10.7 Disabled

- 降低对比度
- 取消 hover 行为
- 保持文本可读
- 如有必要，提供不可操作原因

### 10.8 Loading

- 保持布局稳定
- 页面优先先出骨架，再逐块填充
- 不使用 shimmer skeleton
- 长时间 loading 需要文字说明或进度感知

### 10.9 Error

- 字段级错误要靠近字段显示
- 文案优先说明如何修正
- 页面级错误需有独立反馈区
- 所有错误都不能只靠颜色表达

### 10.10 Success

- 成功反馈应轻量明确
- 可用 toast、inline status、small banner
- 不做庆祝式动效

### 10.11 Empty States

- 需要说明当前为什么为空
- 需要说明下一步可以做什么
- 视觉上保持系统克制感

## 11. 文案语气与内容表现

### 11.1 语气基调

整体文案语气应当：

- 冷静
- 清楚
- 克制
- 有判断力
- 不讨好
- 不浮夸

它应更接近高质量编辑写作与精密产品文案，而不是强营销口号、互联网俏皮语气或模板化后台提示。

### 11.2 官网文案

- 标题短、硬、清楚
- supporting copy 解释价值，不重复标题
- 强调结果与判断，少用空泛形容词
- CTA 文案直接明确

### 11.3 后台文案

- 页面标题准确说明对象与目标
- 描述文字只补充必要上下文
- 操作文案统一使用明确动词
- 状态词表需全系统统一

### 11.4 表单文案

- label 命名具体
- helper text 只解释当前必要信息
- placeholder 不能替代 label
- 错误提示优先告诉用户如何修正
- 多步骤流程每一步都需说明目的

### 11.5 数据表达

- 数字优先可读性
- 单位明确
- 趋势说明和状态说明统一格式
- 不把每个数字都做成重点

## 12. 可访问性与动效降级

### 12.1 可访问性底线

- 正文与关键 UI 文本必须具备足够对比
- 不能仅靠颜色表达状态
- 所有关键控件必须支持键盘操作
- `focus-visible` 必须明显
- 表单标签、错误提示、表格头部需具备语义结构

### 12.2 Reduced Motion

- 去除大位移 reveal
- 去除持续循环装饰动画
- 缩短非必要过渡
- 保留必要状态变化，但减少位移和动感

## 13. 跨项目复用策略

### 13.1 复用时必须保留的核心特征

这些特征决定了系统辨识度，跨项目复用时不建议轻易移除。

#### 视觉骨架

- 深色背景主导
- 低饱和中性色层次
- 高对比文字
- 细边框和分隔线作为主要分层手段
- 大圆角但不软糯

#### 字体分工

- `Newsreader` 用于主标题与关键标题
- `Inter` 用于正文、说明文、输入和数据内容
- `Space Grotesk` 用于标签、表头、元信息、状态信息

#### 强调逻辑

- `#10B981` 只作为局部强调色
- 不做大面积铺色
- 不用强 glow、强光晕、强饱和 neon 风格

#### 交互语言

- hover 轻微上浮或边框增强
- focus 采用低调但明确的 ring
- loading / success / error 保持克制
- 不使用 shimmer、glass、cursor tracking、强 parallax

### 13.2 复用时可以调整的部分

这些部分可以按项目类型做二次适配，但应保持系统气质不变。

#### 页面密度

- 官网可以更宽松
- 后台和数据页可以更中密度
- 文档页和博客首页可以介于两者之间

#### 首屏结构

- 产品官网可以强调价值主张
- 博客首页可以强调作者定位和文章入口
- 控制台首页可以强调 KPI 与最近动作

#### 内容模块类型

- 博客和内容平台：优先列表、归档、系列、目录
- 后台：优先卡片、表格、过滤器、状态区
- 表单流程：优先步骤条、输入区、辅助说明区

#### 强调对象

- 品牌站：强调观点和差异化
- 博客：强调文章与主题结构
- 控制台：强调数据与任务
- 表单：强调当前步骤与下一步动作

## 14. 不同项目类型的适配建议

### 14.1 技术博客

优先保留：

- 文档感导航
- 系列与归档入口
- 内容列表
- 文章详情页的阅读层级

适当弱化：

- 过强 CTA
- 产品介绍式模块

### 14.2 开发者工具官网

优先保留：

- 文档式结构
- 技术标签
- 局部代码预览
- 模块化 feature blocks

适当增强：

- 价值主张
- 功能说明区
- 可信度证明

### 14.3 SaaS 控制台

优先保留：

- 数据层次
- 状态表达
- 表格和列表结构
- 中密度布局

适当弱化：

- 大型 editorial hero
- 过多装饰性留白

### 14.4 内容平台 / 知识库

优先保留：

- 归档体系
- 分类与标签
- 阅读路径
- 内容详情页结构

适当增强：

- 搜索入口
- 系列内容聚合

## 15. 新项目接入流程

### 第一步：确定项目类型

先判断项目更接近：

- 品牌页
- 博客 / 内容平台
- 控制台
- 表单流程

### 第二步：确定密度

选择：

- 宽松
- 中密度
- 中密度偏高

不建议默认走高密度。

### 第三步：接入 Token

优先接入以下内容：

- 颜色体系
- 字体分工
- 间距系统
- 圆角
- 边框
- 阴影
- 动效曲线

### 第四步：选择页面骨架

优先套用现有页面模式，而不是从空白页开始。

### 第五步：再做项目级调整

允许调整：

- 内容顺序
- 页面模块数量
- 页面密度
- 主题词汇

不建议调整：

- 基础 token
- 字体分工
- 交互节奏
- 组件气质

## 16. 复用检查清单

在新项目里套用这套系统前，建议逐项确认：

- 项目是否适合冷静、克制、结构导向的视觉语言
- 是否接受深色底为主的界面风格
- 是否愿意保留 serif + sans + mono 的字体分工
- 是否接受低装饰、低特效的交互风格
- 是否已有页面骨架可以按本系统映射
- 是否能用 token 而不是页面级硬编码来接入

## 17. Do / Don't

### Do

- 用排版和结构建立高级感
- 用系列、归档、目录和列表建立文档感
- 用少量强调色服务信息判断
- 用统一 token 贯穿多页面
- 让品牌感体现在阅读体验和秩序里
- 让官网、后台、内容页、流程页共享同一套视觉语法

### Don't

- 不要使用 glassmorphism
- 不要使用 shimmer border
- 不要使用 spotlight cursor tracking
- 不要依赖 glow 和 blur 作为品牌识别
- 不要把后台做成普通黑色模板
- 不要把官网做成炫技概念稿
- 不要把强调色铺成大面积背景
- 不要让动效比信息更吸引注意力
- 不要让首页像广告页、后台像另一个产品

## 18. 一句话复用策略

如果要把这套规范带到别的项目里，最稳妥的方式不是“照抄页面”，而是：

先复用 Token，再复用组件语气，最后套用页面骨架。

这样不同项目可以拥有不同内容和信息结构，但仍然保持同一种判断力、秩序感和品牌气质。
